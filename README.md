# Telegram-бот для приема заказов на мероприятия

Проект на Python, aiogram 3.x, PostgreSQL и SQLAlchemy async.
Бот не продает билеты напрямую: клиент оставляет заказ, а менеджеры получают его в Telegram-группе.

## Запуск

```bash
cp .env.example .env
# заполните BOT_TOKEN, MANAGERS_CHAT_ID, MANAGER_IDS, DATABASE_URL

docker compose up --build
```

## Миграции

В проект добавлен Alembic. В Docker миграции запускаются автоматически перед стартом бота:

```bash
alembic upgrade head
```

Основные изменения схемы:

- `events.city` — город проведения мероприятия;
- `events.event_type` — тип события, по умолчанию `sport`;
- `events.sport_type` — тип спорта для спортивных мероприятий, по умолчанию `football`;
- `events.is_active` и `events.deleted_at` — мягкое удаление мероприятий;
- `users.first_name`;
- `users.last_name`;
- `users.updated_at`.

Для уже существующих мероприятий `city` заполняется из старого поля `location`, `event_type` — значением `sport`, `sport_type` — значением `football`. После миграции менеджер может отредактировать эти поля через меню.

Опция `RUN_DB_CREATE_ALL=True` сохранена для обратной совместимости со старым способом запуска через `Base.metadata.create_all()`.

## Как пользоваться

### Клиент

1. `/start`
2. Отправить телефон через кнопку Telegram.
3. Клиент сохраняется/обновляется в БД по `telegram_id`.
4. Выбрать город проведения. В списке показываются только города, в которых есть активные мероприятия.
5. Выбрать тип события. В списке показываются только типы, которые реально есть в выбранном городе.
6. Если выбран тип `Спорт`, выбрать тип спорта. В списке показываются только виды спорта, которые реально есть в выбранном городе.
7. Выбрать мероприятие по выбранным фильтрам.
8. Указать количество билетов.
9. Заказ отправляется менеджерам в Telegram-группу.

После регистрации также доступны действия:

- `Купить билет`
- `Связаться с менеджером`
- `Мои заказы`

### Менеджер

1. Указать свой Telegram ID в `MANAGER_IDS`.
2. Перезапустить бота.
3. Написать боту `/manager`.
4. Управлять мероприятиями через кнопки:
   - `Создать мероприятие`
   - `Список мероприятий`
   - `Редактировать мероприятие`
   - `Удалить мероприятие`

При создании и редактировании мероприятия заполняются город, тип события и, если событие спортивное, тип спорта.

### Группа менеджеров

1. Добавьте бота в группу.
2. Укажите ID группы в `MANAGERS_CHAT_ID`.
3. Новые заказы будут приходить в эту группу с inline-кнопками статусов.
4. Для менеджерского меню в группе используйте inline-кнопки. Текстовые команды в группе нужно писать ответом на сообщение бота.

## Примечания

- Удаление мероприятия сделано мягким: событие скрывается из клиентских списков, но старые заказы не ломаются.
- Поле телефона сохранено как `users.phone_number`, чтобы не ломать существующую таблицу и код.
- Для новых изменений БД используйте Alembic, а не ручные правки схемы.
