# Telegram-бот для приема заказов на спортивные мероприятия

Проект на Python, aiogram 3.x, PostgreSQL и SQLAlchemy async.
Бот не продает билеты напрямую: клиент оставляет заказ, а менеджеры получают его в Telegram-группе.

## Запуск

```bash
cp .env.example .env
# заполните BOT_TOKEN, MANAGERS_CHAT_ID, MANAGER_IDS, DATABASE_URL

docker compose up --build
```

## Миграции

В проект добавлен Alembic.

Для существующей базы данных примените миграцию:

```bash
alembic upgrade head
```

Миграция добавляет:

- `events.city` — город проведения мероприятия;
- `users.first_name`;
- `users.last_name`;
- `users.updated_at`.

Для уже существующих мероприятий `city` заполняется из старого поля `location`, чтобы не оставить обязательное поле пустым. После миграции менеджер может отредактировать город у старых мероприятий через меню редактирования.

Опция `RUN_DB_CREATE_ALL=True` сохранена для обратной совместимости со старым способом запуска через `Base.metadata.create_all()`.

## Как пользоваться

### Клиент

1. `/start`
2. Отправить телефон через кнопку Telegram.
3. Клиент сохраняется/обновляется в БД по `telegram_id`.
4. Выбрать город проведения. В списке показываются только города, в которых есть мероприятия.
5. Выбрать мероприятие из выбранного города.
6. Указать количество билетов.
7. Указать свой город или место.
8. Заказ отправляется менеджерам в Telegram-группу.

После регистрации также доступны действия:

- `Купить билет`
- `Связаться с менеджером`
- `Мои заказы`

### Менеджер

1. Указать свой Telegram ID в `MANAGER_IDS`.
2. Перезапустить бота.
3. Написать боту `/manager`.
4. Управлять мероприятиями через кнопки:
   - `Создать мероприятие`
   - `Список мероприятий`
   - `Редактировать мероприятие`
   - `Удалить мероприятие`

При создании и редактировании мероприятия теперь заполняется город проведения.

### Группа менеджеров

1. Добавьте бота в группу.
2. Укажите ID группы в `MANAGERS_CHAT_ID`.
3. Новые заказы будут приходить в эту группу с inline-кнопками статусов.

## Примечания

- Удаление мероприятия запрещено, если по нему уже есть заказы.
- Поле телефона сохранено как `users.phone_number`, чтобы не ломать существующую таблицу и код.
- Для новых изменений БД используйте Alembic, а не ручные правки схемы.

## Автоматические миграции в Docker

При запуске контейнера теперь автоматически выполняется:

```bash
alembic upgrade head
```

Это нужно для существующих PostgreSQL-томов: `Base.metadata.create_all()` не добавляет новые колонки в уже созданные таблицы, поэтому поля `users.first_name`, `users.last_name`, `users.updated_at` и `events.city` должны добавляться миграцией.

Если контейнер уже был собран до этого изменения, пересоберите образ:

```bash
docker compose down
docker compose up --build
```
